نام محصول: خودروبان (KhodroBan)
شعار: کاهش هزینه‌های نگهداری خودرو با هوش مصنوعی - مشاور هوشمند شخصی شما برای مدیریت بهینه خودرو
هدف اصلی: کاربران بتوانند با استفاده از هوش مصنوعی، هزینه‌های نگهداری خودرو خود را کاهش دهند. سیستم با تحلیل الگوی استفاده، سوابق سرویس و هزینه‌ها، پیشنهادات شخصی‌سازی شده برای بهینه‌سازی هزینه‌ها ارائه می‌دهد. همچنین مدیریت خودرو(ها)، ثبت و پیگیری سرویس‌های دوره‌ای، و یادآوری‌های هوشمند برای موعد سرویس را فراهم می‌کند.
مخاطب هدف: مالکان خودرو در ایران که می‌خواهند هزینه‌های نگهداری خودرو خود را کاهش دهند و از مشاوره هوشمند برای مدیریت بهینه خودرو بهره‌مند شوند

# 2. ویژگی‌های کلیدی (Core Features) - فقط MVP

من می‌خواهم یک اپلیکیشن تحت وب (Web App) بسازی که شامل صفحات زیر باشد:

1. صفحه اصلی (Landing Page): شامل معرفی جذاب محصول، مزایای استفاده از خودروبان، دکمه اقدام به عمل (CTA) بزرگ برای ثبت‌نام، و نظرات/تجربیات کاربران.
2. صفحه داشبورد (Dashboard): نمایش لیست خودروهای کاربر با وضعیت سرویس (OK / نزدیک موعد / گذشته از موعد)، کیلومتر فعلی، و هشدارهای فعال. میانبر برای ثبت سریع سرویس یا هزینه.
3. صفحه مدیریت خودروها (Vehicles): لیست خودروها با امکان افزودن، ویرایش و حذف. صفحه جزئیات هر خودرو شامل سوابق سرویس، لیست هزینه‌های روزانه، و امکان به‌روزرسانی کیلومتر.
4. فرم ثبت سرویس/هزینه (Add Service/Expense): یک صفحه واحد با Tab/Toggle برای ثبت سرویس دوره‌ای (تاریخ، کیلومتر، هزینه، نوع سرویس) یا ثبت هزینه روزانه (سوخت، کارواش، پارکینگ، عوارض، تعمیرات).
5. صفحه گزارش‌ها (Reports): نمایش جمع هزینه‌ها با فیلتر بر اساس خودرو، بازه تاریخ و نوع هزینه. امکان خروجی CSV (و PDF برای کاربران Pro).
6. صفحه مشاور هوشمند AI (AI Consultant): یک چت‌بات هوشمند که با استفاده از هوش مصنوعی (Gemini یا OpenAI) به سوالات کاربر درباره مشکلات خودرو، نگهداری، تعمیرات و سرویس‌های دوره‌ای پاسخ می‌دهد. مشاور به اطلاعات خودرو(ها)، سوابق سرویس و هزینه‌های کاربر دسترسی دارد و پیشنهادات شخصی‌سازی شده برای کاهش هزینه‌های نگهداری ارائه می‌دهد. امکان ارسال تصویر برای تشخیص مشکلات خودرو نیز وجود دارد.
7. صفحه تنظیمات (Settings): پروفایل کاربر، تنظیمات یادآوری (فاصله کیلومتر/زمان، روزهای هشدار قبل از موعد)، مدیریت اشتراک Pro، و Export کامل داده‌ها.
8. پنل پاسخ (Response Area): فضایی برای نمایش نوتیفیکیشن‌ها و یادآوری‌های سیستم به کاربر.

# 3. طراحی و ظاهر (Design System)

استایل کلی: مدرن و مینیمال با تم Glassmorphism (شیشه‌ای مات با افکت blur)
رنگ اصلی: گرادیان آبی-بنفش تیره (#1a1a2e → #16213e → #0f3460) برای پس‌زمینه، آبی روشن (#4fc3f7) و فیروزه‌ای (#00bcd4) برای المان‌های اصلی، و نارنجی/طلایی (#ff9800) برای دکمه‌های CTA
فونت: از فونت فارسی Vazirmatn یا یکان با ضخامت‌های مختلف استفاده کن
چیدمان (Layout): کاملاً ریسپانسیو و Mobile-First (مناسب برای موبایل در اولویت)
حس و حال (Vibe): حرفه‌ای، قابل اعتماد و دوستانه - کاربر باید احساس کند که یک ابزار قدرتمند و هوشمند در دست دارد

# 4. منطق فنی و هوشمندسازی (Technical Logic)


**۱. ذخیره‌سازی و همگام‌سازی داده‌ها:**

- تمام داده‌ها (خودروها، سرویس‌ها، هزینه‌ها) در دیتابیس Supabase (PostgreSQL) ذخیره می‌شوند
- Frontend با استفاده از Supabase Client به صورت مستقیم با دیتابیس ارتباط برقرار می‌کند (بدون نیاز به Backend جداگانه)
- داده‌ها به صورت Real-time همگام می‌شوند - هر تغییری بلافاصله در تمام دستگاه‌های کاربر نمایش داده می‌شود
- امنیت داده‌ها با Row Level Security (RLS) در Supabase تضمین می‌شود

**۲. سیستم یادآوری هوشمند:**

- یک سرویس Python (Cron Job) هر روز ساعت ۸ صبح اجرا می‌شود
- این سرویس خودروهای نیازمند سرویس را بررسی می‌کند:
  - آخرین سرویس ثبت‌شده برای هر خودرو را می‌خواند
  - بر اساس `interval_days` (فاصله زمانی سرویس) و `warning_days_before` (روزهای هشدار قبل از موعد) محاسبه می‌کند
  - اگر خودرو در بازه هشدار باشد (مثلاً ۷ روز قبل از موعد)، یک نوتیفیکیشن در جدول `notifications` ایجاد می‌کند
- نوتیفیکیشن‌ها به صورت Real-time در Frontend نمایش داده می‌شوند (بدون نیاز به رفرش صفحه)
- امکان ارسال یادآوری از طریق تلگرام نیز وجود دارد (اگر کاربر ربات تلگرام را متصل کرده باشد)

**۳. محاسبه وضعیت خودروها:**

- برای هر خودرو، سیستم به صورت خودکار وضعیت را محاسبه می‌کند:
  - **OK**: خودرو در وضعیت عادی است و نیازی به سرویس ندارد
  - **نزدیک موعد**: خودرو در بازه هشدار است (مثلاً ۷ روز تا موعد)
  - **گذشته از موعد**: زمان سرویس گذشته است
- این محاسبات بر اساس دو معیار انجام می‌شود:
  - **زمان**: فاصله زمانی از آخرین سرویس
  - **کیلومتر**: کیلومتر طی شده از آخرین سرویس (در صورت فعال بودن)

**۴. مشاور هوشمند AI:**

- سیستم از هوش مصنوعی (Gemini یا OpenAI) برای ارائه مشاوره شخصی‌سازی شده استفاده می‌کند
- مشاور AI به اطلاعات زیر دسترسی دارد:
  - اطلاعات خودرو(ها)ی کاربر
  - سوابق سرویس‌های ثبت‌شده
  - هزینه‌های ثبت‌شده
  - الگوی استفاده از خودرو
- بر اساس این اطلاعات، مشاور پیشنهادات شخصی‌سازی شده ارائه می‌دهد:
  - بهینه‌سازی زمان سرویس برای کاهش هزینه‌ها
  - هشدار درباره هزینه‌های غیرمعمول
  - پیشنهاد راهکارهای کاهش مصرف سوخت
  - تشخیص الگوهای غیرعادی در هزینه‌ها یا سرویس‌ها

**۵. مدیریت هزینه‌ها و گزارش‌گیری:**

- کاربر می‌تواند هزینه‌های روزانه (سوخت، کارواش، پارکینگ، عوارض، تعمیرات) را ثبت کند
- سیستم به صورت خودکار جمع هزینه‌ها را محاسبه می‌کند
- امکان فیلتر بر اساس خودرو، بازه تاریخ و نوع هزینه وجود دارد
- کاربران Pro می‌توانند گزارش‌های کامل را به صورت CSV یا PDF دریافت کنند

**۶. Real-time Updates:**

- تمام تغییرات به صورت Real-time در Frontend نمایش داده می‌شوند
- وقتی Python Cron Job یک نوتیفیکیشن ایجاد می‌کند، کاربر بلافاصله آن را در داشبورد می‌بیند (بدون نیاز به رفرش)
- این قابلیت با استفاده از Supabase Realtime پیاده‌سازی شده است

**۷. امنیت و احراز هویت:**

- احراز هویت با Supabase Auth انجام می‌شود
- هر کاربر فقط به داده‌های خود دسترسی دارد (Row Level Security)
- تمام درخواست‌ها از Frontend با کلید Anon Key انجام می‌شود
- سرویس Python از Service Role Key استفاده می‌کند (فقط برای ایجاد نوتیفیکیشن‌ها)

# 5. محتوا و متن‌ها (Copywriting)

زبان سایت: کاملاً فارسی (RTL - راست به چپ).
لحن نوشته‌ها: خودمونی و دوستانه اما حرفه‌ای - مثل یک مشاور قابل اعتماد که به کاربر کمک می‌کند.
از متن‌های "لورم ایپسوم" استفاده نکن؛ متن‌های واقعی و مرتبط با مدیریت خودرو و نگهداری خودرو بنویس. از اصطلاحات رایج در حوزه خودرو استفاده کن (مثل: سرویس دوره‌ای، تعویض روغن، کیلومتر، موعد سرویس، هزینه‌های نگهداری).
