## مقدمه
من یک اپلیکیشن مدیریت خودرو به نام "خودروبان" (KhodroBan) با استفاده از Cursor توسعه داده‌ام. لطفاً یک مستند فنی جامع و دقیق از این برنامه تولید کن که بتوان از آن برای بازسازی همین برنامه توسط ابزارهای هوش مصنوعی دیگر مانند z.ai، dreamflow.app و stitch.withgoogle.com استفاده کرد.

## دستورالعمل تحلیل

لطفاً کدهای موجود در این پروژه را به دقت تحلیل کن و اطلاعات زیر را استخراج و مستند کن:

### ۱. ساختار پروژه و فایل‌ها
- ساختار کلی پوشه‌ها و فایل‌ها
- نقش هر پوشه و فایل مهم
- وابستگی‌ها و کتابخانه‌های استفاده شده

### ۲. معماری فنی
- الگوی معماری استفاده شده (MVC، MVP، MVVM و...)
- نحوه ارتباط بین کامپوننت‌ها
- جریان داده در برنامه
- مدیریت وضعیت (State Management)

### ۳. تکنولوژی‌ها و فریمورک‌ها
- فریمورک‌های اصلی استفاده شده
- کتابخانه‌های UI و کامپوننت‌ها
- ابزارهای مدیریت داده و API
- ابزارهای استایل‌دهی و تم

### ۴. ویژگی‌ها و قابلیت‌های برنامه
- تمام ویژگی‌های پیاده‌سازی شده
- نحوه عملکرد هر ویژگی
- کامپوننت‌های مرتبط با هر ویژگی
- جریان کار در هر ویژگی

### ۵. مدل‌های داده
- ساختار مدل‌های داده
- روابط بین مدل‌ها
- نوع داده‌ها و محدودیت‌ها
- نحوه ذخیره‌سازی و بازیابی داده‌ها

### ۶. API و ارتباطات سرور
- نقاط پایانی (Endpoints) API
- ساختار درخواست‌ها و پاسخ‌ها
- احراز هویت و مجوزها
- مدیریت خطاها

### ۷. رابط کاربری و تجربه کاربری
- ساختار صفحات و کامپوننت‌ها (با جزئیات کامل)
- لیست تمام صفحات برنامه به همراه توضیحات هر صفحه
- فیلدها و عناصر هر صفحه به همراه نوع و محدودیت‌ها
- کامپوننت‌های قابل استفاده مجدد
- الگوهای طراحی استفاده شده
- تعاملات کاربر و انیمیشن‌ها
- واکنش‌گرایی و طراحی برای دستگاه‌های مختلف

### ۸. طراحی بصری و استایل‌دهی
- تم اصلی و پالت رنگی
- کلاس‌های CSS و کامپوننت‌های استایل
- افکت‌های بصری (مانند Glassmorphism)
- فونت‌ها و تایپوگرافی

### ۹. پیکربندی و تنظیمات
- فایل‌های پیکربندی اصلی
- متغیرهای محیطی
- تنظیمات بیلد و دیپلوی
- پیکربندی ابزارهای توسعه

### ۱۰. نکات فنی و بهینه‌سازی
- بهترین شیوه‌های استفاده شده
- نکات عملکردی و بهینه‌سازی
- ملاحظات امنیتی
- مشکلات شناخته شده و راه‌حل‌ها

## نکات مهم

1. لطفاً تمام جزئیات فنی را با دقت مستند کن تا بتوان برنامه را دقیقاً به همین شکل بازسازی کرد.
2. اگر کدی پیچیده یا نکته فنی خاصی دارد، حتماً آن را توضیح ده.
3. تمام کامپوننت‌ها، توابع و کلاس‌های مهم را مستند کن.
4. اگر از الگوهای طراحی خاصی استفاده شده، آن‌ها را ذکر کن.
5. تمام وابستگی‌ها و نسخه‌های آن‌ها را لیست کن.
6. اگر نکته‌ای برای دیپلوی یا اجرای برنامه وجود دارد، حتماً ذکر کن.
7. **مهم**: در بخش "رابط کاربری و تجربه کاربری"، لطفاً لیست تمام صفحات برنامه را به همراه توضیحات کامل هر صفحه و فیلدهای دقیق هر صفحه (نوع، محدودیت‌ها، اعتبارسنجی و...) ارائه ده.

این مستند فنی باید به گونه‌ای کامل و دقیق باشد که یک توسعه‌دهنده یا ابزار هوش مصنوعی دیگر بتواند با استفاده از آن، برنامه خودروبان را دقیقاً به همین شکل بازسازی کند.
